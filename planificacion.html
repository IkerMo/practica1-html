Esta página describe cómo vamos a organizar el desarrollo del proyecto, qué tareas realizaremos, cómo se repartirán y cuáles serán los hitos principales.

Reparto del trabajo

El desarrollo se ha dividido por páginas para poder trabajar en paralelo y evitar conflictos constantes en el repositorio.

  • Iker: desarrollo completo de la página miembros.html.
  • Carlos: implementación de la página contacto.html y del formulario con mailto.
  • Aaron: desarrollo de index.html, detalles.html y estructura común de navegación (navbar).
  • Xiangzhenhua: planificación, revisión final, validación HTML y preparación del archivo comprimido para la entrega.

Tareas a realizar

  1. Definir el contenido textual del proyecto.
  2. Crear la estructura base HTML5 válida para todas las páginas.
  3. Implementar la barra de navegación común.
  4. Desarrollar las páginas asignadas a cada miembro.
  5. Comprobar que todos los enlaces funcionan correctamente.
  6. Validar cada documento HTML en el validador oficial.
  7. Realizar revisión final y preparar la entrega en formato .zip.

Metodología de trabajo

Se utilizará un repositorio compartido en GitHub para coordinar el desarrollo. Cada miembro subirá sus cambios mediante commits periódicos. Antes de la entrega, se realizará una revisión conjunta para asegurar coherencia en la navegación, limpieza del código e inexistencia de errores de validación.
